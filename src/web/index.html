Complete reference for the Clinic Appointment Booking System API. All endpoints, request bodies, responses, and authentication requirements.

ℹ️
Authentication: All protected endpoints require Authorization: Bearer <accessToken> header. The accessToken is returned in the response body and should be stored in memory only (not localStorage). The refreshToken is set as an HttpOnly cookie automatically.
⚠️
Error Format: All errors follow { "success": false, "error": { "code": "ERROR_CODE", "message": "...", "details": [...] } }

Authentication
/api/v1/auth
POST
/auth/register
Register a new user. Sends OTP to email for verification. Optionally upload a profile photo.
Public multipart/form-data
▼
Request Body — multipart/form-data
{
  "name":     "Ahmed Ali",           // required, 2–100 chars
  "email":    "ahmed@example.com",   // required, unique
  "password": "Pass1234",            // required, min 8, needs uppercase + number
  "role":     "patient",             // optional: "patient" | "doctor" — default: "patient"
  "image":    <file>                 // optional, image only, max 5MB
}
Response 201
{
  "success": true,
  "data":    "Verification OTP sent to email"
}
Possible Errors
409EMAIL_TAKEN — Email already registered
422VALIDATION_ERROR — Invalid fields with details array
💡
After register → call POST /auth/verify-email with the OTP from email to get your access token.

GET
/auth/google
Redirect user to Google OAuth consent screen. On success, redirects to CLIENT_URL/auth/google/success?token=ACCESS_TOKEN
Public
▼
🔗
Flow:
1
Frontend redirects user to GET /auth/google
2
Google consent screen → user approves
3
Backend callback → creates/updates user → issues tokens
4
Redirects to CLIENT_URL/auth/google/success?token=ACCESS_TOKEN
5
Frontend reads ?token from URL and stores in memory
⚠️
On failure, redirects to /api/v1/auth/google/error which returns 401 JSON response.

Payments
/api/v1/payments — Stripe EGP
POST
/payments/create-intent/:appointmentId
Create a Stripe PaymentIntent for a pending appointment. Returns clientSecret to complete payment on frontend.
Patient Only
▼
💳
Payment Flow:
1
Call this endpoint → get clientSecret
2
Use Stripe.js on frontend: stripe.confirmCardPayment(clientSecret)
3
Stripe calls our webhook POST /payments/webhook
4
Webhook confirms appointment → status: confirmed, paymentStatus: paid
5
WebSocket fires appointment:confirmed event to patient
Response 200
{
  "success": true,
  "data": {
    "clientSecret": "pi_3Ox...secret_..."  // pass to Stripe.js
  }
}
Possible Errors
400INVALID_STATUS — Appointment is not pending
403FORBIDDEN — Not your appointment
POST
/payments/webhook
Stripe webhook handler. Requires raw body. Verifies signature. Do NOT call manually.
Stripe Only
▼
🔒
This endpoint is for Stripe only. It requires the raw request body and validates the stripe-signature header. Configure your webhook URL in the Stripe dashboard.
Handled Events
payment_intent.succeeded:
  → appointment.status = "confirmed"
  → appointment.paymentStatus = "paid"
  → Remove BullMQ expiry job
  → Invalidate slots cache
  → Emit WS: appointment:confirmed
  → Send confirmation email
  → Schedule 24h reminder

payment_intent.payment_failed:
  → No action (appointment stays pending until BullMQ expires it)
POST
/payments/refund/:appointmentId
Manual full refund via Stripe. Only for paid appointments.
Admin Only
▼
Response 200
{ "success": true, "message": "Refund processed successfully" }
Possible Errors
400NOT_PAID — Appointment was not paid

⚡
WebSocket Events
Socket.io — Real-time notifications
🔌
Connect: Pass the access token in the handshake auth:
const socket = io(BASE_URL, { auth: { token: accessToken } })
⚠️ No "Bearer" prefix — raw JWT only
Server → Client Events
{
  "appointment:created":   { appointmentId, doctorId, date, startTime },
  "appointment:confirmed": { appointmentId },   // payment succeeded
  "appointment:cancelled": { appointmentId },   // by any party
  "appointment:completed": { appointmentId },   // doctor marked done
  "appointment:expired":   { appointmentId },   // payment not made in time
  "appointment:reminder":  { appointmentId, date, startTime }  // 24h before
}
// All events are scoped to the authenticated user's room: user:{userId}
❌
Error Codes Reference
All errors follow { success: false, error: { code, message, details? } }
{
  // Auth
  "UNAUTHORIZED":          401  // No token / invalid credentials
  "INVALID_TOKEN":          401  // Malformed JWT
  "TOKEN_EXPIRED":          401  // Access token expired → call /refresh
  "REFRESH_TOKEN_REUSE":    401  // Reuse detected → re-login required
  "FORBIDDEN":              403  // Authenticated but insufficient role
  "EMAIL_NOT_VERIFIED":     403  // Must verify email before login
  "EMAIL_TAKEN":            409  // Email already registered
  "ALREADY_VERIFIED":       400  // Email already verified
  "OTP_EXPIRED":            400  // OTP expired (10 min window)
  "INVALID_OTP":            400  // Wrong OTP code

  // Resources
  "NOT_FOUND":              404  // Resource not found
  "DUPLICATE_KEY":          409  // MongoDB unique constraint
  "VALIDATION_ERROR":       422  // Zod/Mongoose validation (details array)
  "INVALID_ID":             400  // Invalid MongoDB ObjectId

  // Appointments
  "DOCTOR_UNAVAILABLE":     400  // Not verified or not active
